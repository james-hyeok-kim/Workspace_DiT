### Model and Architecture
* **Model:** PixArt-Σ, a DiT-based (Diffusion Transformer) model.
* **Architecture Details:** It uses 28 attention blocks that include self-attention, cross-attention, and feed-forward layers.
* **Parameter Count:** 600M parameters.

### Dataset and Calibration
* **Calibration Dataset:** A random sample of prompts from COCO Captions 2024 is used for calibration.
* **Main Benchmark Evaluation (Performance Comparison):** A total of 10,000 prompts were sampled to generate images to evaluate the overall performance and generalization ability of each model and quantization method.
    * **Artistic Image Generation Evaluation:** 5,000 prompts were randomly extracted and used from the MJHQ-30K dataset.
    * **Realistic Image Generation Evaluation:** 5,000 prompts were randomly extracted and used from the sDCI (summarized Densely Captioned Images) dataset.

### Inference Settings
* **Steps:** Evaluated using the default 20-step setting.

### Evaluation Metrics

#### 1. Image Quality and Visual Fidelity
Evaluates how natural the generated images are and their overall quality.
* **FID (Fréchet Inception Distance) [↓ Lower is better]:** Measures the distance between the distribution of the generated images and the distribution of actual (Ground-truth) images. It is the most standard metric used in diffusion model evaluation; a lower value indicates higher quality and greater similarity to real photos.
* **Image Reward [↑ Higher is better]:** The score of a reward model trained to approximate 'Human Preference' for the output of text-to-image generation models. It quantifies how well the generated image matches the prompt and how visually satisfying it is to humans.
* **CLIP IQA (Image Quality Assessment) [↑ Higher is better]:** A metric that evaluates the overall quality (e.g., sharpness, naturalness) of an image utilizing the visual feature extraction capabilities of the CLIP model.

#### 2. Similarity to Original 16-bit Models
Because this is a quantization study, the core focus is on how identically the 4-bit compressed model reproduces the output of the original FP16/BF16 model without errors.
* **LPIPS (Learned Perceptual Image Patch Similarity) [↓ Lower is better]:** Calculates the 'perceptual difference' perceived by humans using deep learning network feature maps. It measures the structural and semantic differences between the images generated by the 16-bit original model and the 4-bit model. A value closer to 0 means it appears visually identical to the original.
* **PSNR (Peak Signal-to-Noise Ratio) [↑ Higher is better]:** A metric that measures the numerical similarity at the pixel level between two images. It indicates how minimal the noise (error) introduced by the quantization process is.
* **SSIM (Structural Similarity Index Measure) [↑ Higher is better]:** Comprehensively evaluates the similarity of luminance, contrast, and structure between the two images—beyond simple pixel value differences—to determine how well the details of the original image are preserved.

#### 3. Text-Image Alignment
* **CLIP Score [↑ Higher is better]:** Measures the cosine similarity between the input text prompt and the generated image using the CLIP model. It demonstrates how accurately the model reflects the user's text instructions when generating the image, even after quantization.
